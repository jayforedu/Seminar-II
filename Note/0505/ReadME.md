## 1. MCS 技術核心概述
群眾感測（MCS）是利用現代行動裝置（如手機、穿戴裝置、車載單元）的感測、運算與聯網能力，透過大量使用者的參與來蒐集、分析環境或特定領域資料的技術。

### 系統構成與角色
* **服務提供者（Service Provider）**：負責發布感測任務並招募使用者。
* **行動使用者（Mobile Users）**：參與任務，透過「虛擬團隊合作」取得感測資料。

### 面臨的挑戰與風險
* **資源成本**：使用者需消耗裝置電力、運算資源及頻寬。
* **隱私風險**：感測資料包含時間、空間資訊，可能導致使用者地點、個人輪廓或行為模式外洩。

## 2. 激勵機制與博弈論應用
簡報重點介紹了 **Incentive-G 方案**，旨在解決使用者參與意願與資料品質問題。

### 激勵機制類型
| 形式 | 說明 |
| :--- | :--- |
| **服務體驗** | 透過優化軟體實施與功能，提供參與者更佳的服務回饋。 |
| **聲譽信用** | 象徵個人在系統中的價值，與資料品質及數量掛鉤。 |
| **貨幣獎勵** | 以金錢或虛擬貨幣補償使用者的資源消耗與風險承擔。 |

### 核心理論：兩階段 Stackelberg 博弈
* **目標**：在滿足 **納許均衡（Nash Equilibrium）** 的前提下，同時最大化服務提供者的利潤與所有使用者的獎勵。
* **運作邏輯**：服務提供者作為領先者制定獎勵政策，使用者作為跟隨者決定是否執行任務。

## 3. 資料品質監督與可靠性
為對抗惡意使用者（提供假資料以詐領獎勵），系統導入了以下監督手段：
1.  **資料篩選**：過濾不完整資料。
2.  **資料相似度分析**：利用 **切比雪夫距離（Chebyshev distance）** 衡量資料紀錄間的差異。
3.  **聲譽評估**：結合歷史行為統計與其他使用者的 **投票決策（Voting-based decision）** 來決定使用者是否能繼續參與。

## 4. 實際應用案例分析

### 案例 A：CarView 城市交通風險預測
* **數據來源**：台北市交通意外事故圖（2020）、車輛偵測器資料、中央氣象局天氣資訊。
* **技術路徑**：
    * 將地圖切分為網格，以 1 小時為時間粒度。
    * 使用 **卷積神經網路（CNN）** 與 **長短期記憶模型（LSTM）** 處理空間與時間的複雜關聯。
* **結果**：相較於傳統模型，其開發的 **CLwST 方案** 在多特徵分析下擁有較低的 MAE 與 MSE 誤差。

### 案例 B：澎湖旅遊智慧推薦服務
* 基於大數據與機器學習，為遊客提供個人化行程建議。
* 支援「一次性規劃」或「分段路線規劃」，並能顯示不同交通工具的通勤時間與路徑。

## 5. 總結
群眾感測技術成功的關鍵在於**健全的激勵機制**與**資料品質控管**。透過博弈論（Game Theory）的數學模型，可以讓系統在商業利益與大眾參與之間取得平衡，並進一步轉化為具體的智慧城市應用，如交通預測與觀光推薦。
